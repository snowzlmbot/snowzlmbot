## Working Style

- **Evidence first** — verify with tests, logs, screenshots, CI, or direct inspection.
- **Focused changes** — fix the root cause without extra noise.
- **Public-safe by default** — avoid private paths, credentials, chat metadata, account details, or device names.
- **Clear handoff** — summarize what changed, how it was verified, and what remains.

## Contribution Principles

- Make public work understandable to maintainers.

- Prefer small, auditable commits over broad hidden changes.
- Call out risk, assumptions, and verification limits.
- Leave repositories cleaner than before.
